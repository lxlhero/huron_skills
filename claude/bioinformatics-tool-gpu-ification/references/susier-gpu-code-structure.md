## Files

### src/susieR_gpu.py (452 lines)
Path: `transfer2gpu/susieR/src/susieR_gpu.py`

GPU-accelerated SuSiE regression. All float64, all GPU-resident during IBSS loop.

Functions:
- `compute_Xb_gpu(X, b)` — `torch.mv(X, b)` matrix-vector multiply
- `compute_Xty_gpu(X, y)` — `torch.mv(X.T, y)` crossprod
- `compute_column_variances(X)` — `torch.sum(X*X, dim=0)` for d_j
- `single_effect_regression_gpu(Xr, d, sigma2, V, prior_weights)` — SER step:
  - s2 = V * sigma2 / (V * d + sigma2)
  - lbf = -0.5 * log1p(V*d/sigma2) + 0.5 * Xr^2 * s2 / sigma2^2
  - alpha = F.softmax(lbf, dim=0)
  - mu = s2 * Xr / sigma2, mu2 = mu^2 + s2
- `estimate_prior_variance_gpu(alpha, mu, mu2)` — EM update: V = sum(alpha * mu2)
- `_update_sigma2(y, Xb_total, alpha, mu, mu2, d, sigma2, V, n)` — residual variance with correction term
- `susie_gpu(X, y, L, max_iter, tol, ...)` — full IBSS loop:
  1. Precompute d = colSums(X^2) and Xty = X^T y
  2. Initialize alpha/mu/mu2/lbf as L×p zero tensors
  3. For each iteration: for each effect l, remove contribution → compute Xr → SER → add new contribution
  4. Optional EM updates for sigma2 and V
  5. Convergence on max|alpha_change| < tol
  6. Return dict with all numpy arrays
- `susie_gpu_numpy(X_np, y_np, L, ...)` — entry point for reticulate:
  - numpy → `torch.from_numpy(np.ascontiguousarray(...))` → GPU
  - Auto-detects CUDA, warns if unavailable
  - Returns dict of numpy arrays

### scripts/susieR_gpu_wrapper.R (326 lines)

R wrapper for GPU-accelerated SuSiE. API mirrors `susie::susie()`.

## Delegation Pattern

For bioinformatics GPU-ification projects with R tools:
1. **Claude Code (via delegate_task)**: Technical plan + GPU kernel code + R wrapper + benchmark data sources
2. **Hermes (direct ops)**: Docker build, push, rjob submission, result collection, progress tracking

This division works because:
- GPU code is reasoning-heavy (correctness, numerical stability, algorithm translation)
- Ops are mechanical but require awareness of environment state (build progress, registry, GPFS paths)
- Claude has no session memory — it needs full context injected
- Hermes tracks the session's operational state (which job is running, what file is where)
